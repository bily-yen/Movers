Overview

This tool computes the geoid height and orthometric height of a point (or points).

User-specified inputs include:

  1. The Earth Gravitational Model to compute the geoid. The number following EGM represents the model year, the number following the dash denotes the model grid spacing in minutes of a degree.
  2. The option for cubic interpolation between grid points. Otherwise no interpolation is performed. See the GeographicLib information pages for more info on the interpolation method.
  3. The latitude, longitude, and height of a point(s) above the WGS84 reference ellipsoid.

Understanding the geoid and "height"

The geoid is one approximation of Earth's shape due to gravity, roughly coinciding with mean sea level. Because the inside of our planet is non-uniform, gravity is not constant at the surface. If Earth were completely covered in an ocean, the water would swell over gravity highs (where there is more mass below the surface) and depress in areas of gravity lows (where there is less mass below the surface). The geoid approximates this hypothetical ocean surface, representing a geopotential surface. To think of it another way, if we were able to walk on the geoid, we would experience a constant value of gravity.

Representation of the geoid, image from NASA.

The height of the geoid is often reported relative to the internationally recognized WGS84 reference ellipsoid. The GPS in our phones, for example, records elevations relative to this ellispoid. Sometimes, it is useful to know elevations relative to mean sea level—this is where the geoid height calculation comes in: to convert ellipsoidal heights to heights above the geoid (aka orthometric heights). The NGA (National Geospatial-Intelligence Agency) publishes models that describe the earth's gravitational field (see the Earth Gravitational Models here), and updates them as we learn more through satellite geodesy. It is becoming standard to use EGM2008 over its predecessors, but a 2020 model is soon to become standard.

Terminology definitions:

  • Reference Ellipsoid: Here, the reference ellipsoid refers to the datum surface derived from the WGS84 reference system.
  • Geoid: The gravitational equipotential surface that approximates global mean sea level, given as the height above the reference ellipsoid.
  • Geoid Undulation: Height of the geoid above the reference ellipsoid.
  • Geoid Height: Same as geoid undulation.
  • Orthometric Height: Height above the geoid.

Understanding the geoid calculation

Computing the height of a geoid from a set of potential coefficients provided in an Earth Gravitational Model (EGM), as done here, is a non-unique problem, dependent on several assumptions. If you'd like to know more about the calculation, a core paper that outlines the methodology can be found here: Use of Potential Coefficient Models for Geoid Undulation Determination Using a Spherical Harmonic Representation of the Height Anomaly/Geoid Undulation Difference by R.H. Rapp, Journal of Geodesy, 1996.

Further discussion of the assumptions in computing a geoid from an EGM is presented in There is no such thing as "The" EGM96 geoid: Subtle points on the use of a global geopotential model, by Dru A. Smith, 1998.

A good description of errors in the geoid height calculations, including those introduced by the cubic interpolation method, can be found on the GeographicLib page. In sum, the errors from the calculations presented here (when compared against the NGA geoid test datasets) stem from (1) the method of interpolation, (2) the height ambiguity resulting from the fact that the geoid and ellipsoid are not parallel surfaces, and (3) the grid format of the geoid harmonic coefficients, which require quantization of height to 3 mm.

A comparison of errors from this calculator with the NGA test datasets is performed in a companion Notebook created by Brooks Mershon and Cassie Hanagan at UNAVCO. An associated map highlights areas where the geoid height calculations differ the most.

Code origin

The original UNAVCO tool ran on a NGA (National Geospatial-Intelligence Agency) fortran code for the geoid height calculation. The original tool can be found here.

The geoid height calculation was recoded in C++ as part of a larger library of geodetic functions in GeographicLib. The GeographicLib::Geoid class and associated Geoid.cpp were translated to JavaScript by Kim Vandry, available here. All products are under the MIT License.

This notebook is run with core functions from another Notebook created by Brooks Mershon and Cassie Hanagan at UNAVCO, which makes Kim Vandry's JavaScript port of GeographicLib::Geoid available in Observable Notebooks like this one. That notebook includes a suite of test cases and further documentation to validate the accuracy of this calculator.
